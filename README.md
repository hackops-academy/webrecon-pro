## рҹ”Қ Overview

**WebRecon Pro** is a professional-grade, all-in-one web penetration testing framework written in Python. Unlike single-purpose tools like Nikto or WPScan, WebRecon Pro is a **complete workflow** вҖ” it chains recon, scanning, and exploitation detection into one seamless pipeline, saving state between runs and generating professional HTML reports.

Built for **Kali Linux**, it uses fully asynchronous HTTP requests for maximum speed and supports multithreading for large-scale subdomain enumeration.

---

## вңЁ Features

### рҹҢҗ Subdomain Enumeration
- Async DNS brute-force with a built-in wordlist of 100+ common subdomains
- Certificate Transparency log harvesting via **crt.sh**
- Live host detection with HTTP status checking
- Custom wordlist support (compatible with SecLists)
- IP resolution for all discovered hosts

### рҹ”Һ Web Fingerprinting
- **CMS Detection:** WordPress, Drupal, Joomla, Magento, Shopify
- **Framework Detection:** Laravel, Django, Rails, React, Angular, Vue.js, Next.js, Nuxt.js
- **WAF Detection:** Cloudflare, Akamai, Imperva/Incapsula, Sucuri, ModSecurity, F5 BIG-IP, Barracuda, Fortinet
- **Sensitive Path Discovery:** `.env`, `.git/config`, `wp-config.php`, `phpinfo.php`, backup files, admin panels, API docs, and 40+ more

### рҹ’Ҙ Vulnerability Scanning
- **SQL Injection** вҖ” GET, POST, and cookie parameters with error-based and time-based detection
- **Cross-Site Scripting (XSS)** вҖ” Reflected XSS with 14+ payloads including filter bypasses
- **Open Redirect** вҖ” Tests 15 redirect parameters with 10 bypass payloads
- **SSRF** вҖ” AWS metadata, GCP metadata, internal network probing
- **Path Traversal** вҖ” Unix and Windows file system traversal with encoding bypasses
- **Command Injection** вҖ” Shell metacharacter injection with output detection
- **CORS Misconfiguration** вҖ” Arbitrary origin reflection, null origin, credentials bypass
- **Clickjacking** вҖ” X-Frame-Options and CSP frame-ancestors validation

### рҹӣЎпёҸ Security Headers Analysis
- Checks 10 critical security headers with severity scoring
- Detects dangerous information disclosure headers
- Cookie security flag validation (HttpOnly, Secure, SameSite)
- Generates a **security score out of 100** with a letter grade (AвҖ“F)
- Full remediation guidance for every missing header

### рҹ”җ Authentication Testing
- **Default credential testing** with 20 common username/password combinations
- **JWT security analysis** вҖ” algorithm none attack, weak secret brute-force, missing expiration, sensitive data in payload
- **Session security** вҖ” entropy checking, predictable session ID detection
- **Brute force protection** вҖ” rate limiting and CAPTCHA detection
- Supports form-based and JSON-based login endpoints

### рҹ”Ң API Security Testing
- **Swagger/OpenAPI spec discovery** вҖ” auto-detects exposed API documentation
- **IDOR testing** вҖ” enumeration of ID-based endpoints for unauthorized access
- **GraphQL security** вҖ” introspection enabled, sensitive types, playground exposure
- **Mass assignment** вҖ” tests for privilege escalation via field injection
- **Broken API authentication** вҖ” unauthenticated access, invalid token acceptance
- **Verbose error detection** вҖ” stack traces, SQL errors, internal paths in responses

### рҹ“Ҡ Professional Reporting
- **HTML report** вҖ” dark-themed, professional security report with severity-colored findings
- **JSON export** вҖ” machine-readable for integration with other tools
- **Plain text report** вҖ” simple output for quick review
- All scans stored in **SQLite database** for history and comparison

## вҡҷпёҸ Installation

### Prerequisites

- Kali Linux (recommended) or any Debian/Ubuntu-based system
- Python 3.9 or higher
- pip3

### Step 1 вҖ” Clone or create the tool folder

```bash
mkdir ~/web-scanner && cd ~/web-scanner
```

### Step 2 вҖ” Set up the folder structure

```bash
mkdir -p modules utils
touch modules/__init__.py utils/__init__.py
```

### Step 3 вҖ” Install dependencies

```bash
pip3 install typer rich httpx beautifulsoup4 --break-system-packages
```

### Step 4 вҖ” Verify installation

```bash
python3 main.py --help
```

You should see the WebRecon Pro banner and command menu.

### Optional вҖ” Install SecLists for better wordlists

```bash
sudo apt install seclists -y
# Wordlists will be at: /usr/share/seclists/Discovery/DNS/
```

## рҹҡҖ Usage

### Full Scan

Run all modules against a target in one command:

```bash
python3 main.py scan https://target.com
```

**With options:**

```bash
python3 main.py scan https://target.com [OPTIONS]

Options:
  -o, --output    PATH      Output directory for reports     [default: ./reports]
  -t, --threads   INT       Number of concurrent threads     [default: 10]
  -w, --wordlist  PATH      Custom subdomain wordlist
  -f, --format    TEXT      Report format: html, json, txt   [default: html]
  -v, --verbose             Show detailed output
  --skip-subdomains         Skip subdomain enumeration
  --skip-vuln               Skip vulnerability scanning
  --skip-auth               Skip authentication testing
  --skip-api                Skip API security testing
```

---

### Subdomain Enumeration

```bash
python3 main.py subdomains <DOMAIN> [OPTIONS]

Options:
  -t, --threads   INT   Concurrent threads          [default: 20]
  -w, --wordlist  PATH  Custom wordlist file
  -o, --output    PATH  Save results to file
  -v, --verbose         Show each resolution attempt
```

**Example:**
```bash
python3 main.py subdomains target.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -t 50 -o subs.txt
```
